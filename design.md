# Design: 픽미업! (Pick Me Up!)

> **문서 목적:** `PRD.md`를 구현하기 위한 제품 디자인 가이드  
> **제품명:** 픽미업!  
> **기준 화면:** 초등 교실 전자칠판, 빔프로젝터, 교사용 노트북  
> **참고 무드:** 업로드된 3D 클레이 AI 로봇 이미지 2종, 기존 네온 픽셀 아케이드 무드보드

---

## 1. 디자인 방향

픽미업!은 초등 교실에서 선생님이 반 명단을 한 번 입력해 여러 뽑기 게임을 실행하는 **교실용 추첨 포털**이다. 화면은 아이들이 함께 볼 만큼 즐겁고, 선생님이 수업 중 빠르게 조작할 만큼 명확해야 한다.

핵심 인상은 다음과 같다.

> **"AI 로봇 친구가 도와주는 밝고 신나는 교실 뽑기 스테이션"**

기존 레퍼런스의 네온 픽셀 아케이드 감성은 그대로 복사하지 않고, 픽미업!에서는 **소프트 3D 로봇 + 깨끗한 교실 UI + 라임/보라 포인트**로 재해석한다.

---

## 2. 메인 캐릭터

### 2.1 캐릭터 정체성

업로드된 이미지의 로봇을 픽미업!의 메인 캐릭터로 사용한다.

| 항목 | 방향 |
|---|---|
| 역할 | 뽑기 도우미, 진행자, 결과 발표자 |
| 성격 | 친절함, 장난기, 공정함, 수업을 방해하지 않는 밝음 |
| 형태 | 흰색 둥근 바디, 검은 얼굴 디스플레이, 보라색 헤드셋/관절 |
| 포인트 | 라임색 눈/안테나/가슴 링, 보라색 태블릿, 노란 전구 |
| 표정 | 웃는 눈, 작은 미소, 놀람, 축하, 생각 중 |

### 2.2 이미지 레퍼런스

- `ChatGPT Image 2026년 7월 7일 오후 04_37_33.png`  
  교실 안내자 장면. 체크리스트, 칠판, 학생, 질문 말풍선이 있어 Hub/도움말/빈 상태에 적합하다.

- `ChatGPT Image 2026년 7월 7일 오후 04_37_51.png`  
  대표 캐릭터 컷. 스마트 기기, 전구, 책, 연필, 라임/핑크 장식이 있어 로비, OG 이미지, 앱 아이콘, 결과 축하에 적합하다.

### 2.3 캐릭터 사용 위치

| 화면 | 사용 방식 |
|---|---|
| Hub | 로봇이 명단 입력을 안내하거나 체크리스트를 들고 있음 |
| Lobby | 로봇이 게임 카드를 소개하는 진행자 |
| Game Intro | 로봇이 "준비됐나요?" 상태로 짧게 등장 |
| Result | 로봇이 당첨자 옆에서 축하 |
| Empty State | 로봇이 빈 명단/저장 없음/불러오기 실패를 부드럽게 안내 |
| Error | 걱정 표정 로봇 + 짧은 해결 문구 |

### 2.4 캐릭터 금지 규칙

- 이미지 안에 한글 텍스트를 직접 넣지 않는다.
- 로봇 얼굴을 복잡하게 만들지 않는다. 눈 2개와 입 1개 중심으로 유지한다.
- 무서운 기계, 전투형 로봇, 과한 SF 느낌을 피한다.
- 캐릭터가 교사용 입력 UI를 가리지 않게 한다.

## 4. 디자인 원칙

### 4.1 교사는 빠르게, 학생은 즐겁게

교사용 화면은 설정이 많아도 조작 순서가 분명해야 한다. 학생용 화면은 결과를 기다리는 시간이 즐거워야 한다.

### 4.2 게임보다 수업 흐름이 우선

모션, 사운드, 장식은 수업 흐름을 방해하지 않아야 한다. 선생님은 언제든 멈추고, 다시 하고, 결과를 복사할 수 있어야 한다.

### 4.3 공정성이 보여야 한다

제외 학생, 가중치, 중복 허용, 뽑힌 사람 제외 여부가 화면에서 분명해야 한다.

### 4.4 전자칠판에서 읽혀야 한다

학생 이름, 당첨자, 시작 버튼, 결과 버튼은 3m 거리에서도 읽혀야 한다.

## 5. 게임 범위 추천

### 5.1 MVP 추천

| 단계 | 게임 | 이유 |
|---|---|---|
| Phase 1 | 제비뽑기 | 발표자 1명, N명, 전원 순서를 모두 처리하는 가장 실용적인 게임 |
| Phase 1 | 돌림판 | 전자칠판에서 보는 맛이 강하고 대표 화면으로 쓰기 좋음 |
| Phase 1.5 | 사다리타기 | 역할 나누기와 모둠 배정에 유용하지만 구현 난도가 조금 높음 |

### 5.2 후순위 추천

| 단계 | 게임 | 이유 |
|---|---|---|
| Phase 2 | 레이스 | 몰입감은 높지만 물리/순위/연출 구현이 큼 |
| Phase 3 | 캡슐 뽑기 | 이벤트성 재미는 좋지만 슬롯/가챠의 도박 연상을 줄여야 함 |

### 5.3 네이밍 가이드

`슬롯/가챠`는 학교 맥락에서 직접적인 도박 연상이 있을 수 있으므로 UI에서는 **캡슐 뽑기**를 기본 명칭으로 사용한다.

## 6. 컬러 시스템

픽미업!의 색은 보라색과 라임을 브랜드 포인트로 쓰되, 전체 화면을 어둡게 몰아가지 않는다. 설정 화면은 밝고, 게임 화면은 조금 더 화려하게 간다.

```css
:root {
  /* Brand */
  --pick-purple-950: #22005C;
  --pick-purple-800: #4B16B8;
  --pick-purple-600: #7551F2;
  --pick-lime-400: #BFFF22;
  --pick-lime-300: #DFFF64;
  --pick-pink-400: #FF6FCF;
  --pick-yellow-400: #FFD84A;
  --pick-mint-400: #73F7C5;

  /* Surface */
  --paper: #FFFDF8;
  --surface: #FFFFFF;
  --surface-lavender: #F3EDFF;
  --surface-pink: #FFF0F8;
  --surface-lime: #F4FFD0;

  /* Text */
  --ink: #171320;
  --ink-purple: #32126A;
  --muted: #6F6484;
  --on-dark: #FFFFFF;

  /* Feedback */
  --success: #3CD278;
  --warning: #FFB02E;
  --danger: #FF5C7A;

  /* Effects */
  --glow-lime: rgba(191, 255, 34, 0.45);
  --glow-purple: rgba(117, 81, 242, 0.38);
  --shadow-soft: rgba(31, 12, 75, 0.18);
}
```

### 6.1 사용 비율

| 용도 | 색상 | 비율 |
|---|---:|---:|
| 배경/넓은 면 | 화이트, 연보라, 딥 퍼플 | 55% |
| 카드/입력 영역 | 화이트, 페이퍼 | 25% |
| 주요 CTA | 라임 | 8% |
| 보조 강조 | 핑크, 노랑, 민트 | 8% |
| 텍스트/라인 | 잉크, 보라 | 4% |

### 6.2 화면별 배경

| 화면 | 배경 |
|---|---|
| Hub | 밝은 연보라 그라디언트 + 흰 카드 |
| Lobby | 보라 계열 배경 + 게임 카드 |
| Game | 게임별 배경, 중앙 액션 영역 강조 |
| Result | 밝은 축하 배경 + 큰 결과 카드 |
| Fullscreen | 딥 퍼플 또는 게임별 몰입 배경 |

## 8. 공통 레이아웃

### 8.1 Header

```txt
[픽미업!] [우리 반 이름]                         [소리] [전체화면]
```

규칙:

- 제품명은 왼쪽에 고정한다.
- 반 이름은 결과 저장 이미지에도 이어진다.
- 전체화면, 소리, BGM, 처음으로는 아이콘+짧은 텍스트로 제공한다.

### 8.2 App Shell

```txt
┌────────────────────────────────────────────┐
│ Header                                     │
├────────────────────────────────────────────┤
│ Main Content                               │
│                                            │
│ Primary Action Area                        │
│                                            │
└────────────────────────────────────────────┘
```

규칙:

- 카드 안에 카드를 중첩하지 않는다.
- 설정/결과처럼 정보가 많은 화면은 밝은 카드 중심으로 구성한다.
- 게임 화면은 큰 액션 영역과 보조 패널을 분리한다.

## 9. 화면별 디자인

## 9.1 Hub: 반 설정

목표: 선생님이 반 이름과 명단을 빠르게 입력하고 바로 게임으로 이동한다.

```txt
┌─────────────────────────────────────────────┐
│ Header                                      │
├───────────────────────┬─────────────────────┤
│ 로봇 안내 영역          │ 반 설정 카드          │
│ - 메인 캐릭터           │ - 반 이름             │
│ - 짧은 안내 문구        │ - 명단 textarea       │
│ - 자동 저장 상태        │ - 제외 체크           │
│                       │ - 게임 고르기 버튼     │
└───────────────────────┴─────────────────────┘
```

디자인:

- 왼쪽 로봇은 크게 보이되 입력 카드보다 강하지 않게 배치한다.
- 명단 입력창은 매우 넓게 제공한다.
- 자동 저장 상태는 카드 하단에 작고 안정적으로 표시한다.
- 위험 버튼인 "저장된 데이터 지우기"는 기본 흐름에서 떨어뜨려 배치한다.

문구 예:

- "명단을 붙여넣으면 바로 뽑을 수 있어요."
- "자동 저장됨"
- "저장 실패"
- "저장된 데이터 지우기"

## 9.2 Lobby: 게임 선택

목표: 선생님이 상황에 맞는 뽑기 게임을 고른다.

```txt
┌─────────────────────────────────────────────┐
│ 오늘은 어떤 뽑기를 할까요?                    │
│ [1명] [여러 명] [순서] [역할] [재미]           │
├──────────┬──────────┬──────────┬───────────┤
│ 제비뽑기  │ 돌림판    │ 사다리    │ 준비 중     │
│ 빠른 발표 │ 두근두근  │ 역할 나누기│ 레이스/캡슐 │
└──────────┴──────────┴──────────┴───────────┘
```

게임 카드 구성:

- 게임 아이콘 또는 미니 일러스트
- 게임명
- 한 줄 설명
- "이럴 때" 태그
- 시작 버튼

추천 카드 순서:

1. 제비뽑기
2. 돌림판
3. 사다리타기
4. 레이스
5. 캡슐 뽑기

## 9.3 제비뽑기

목표: 가장 빠르게 발표자 또는 순서를 뽑는다.

```txt
┌───────────────────────────────┬─────────────┐
│ 쪽지/상자 연출 영역             │ 설정 패널     │
│                               │ 몇 명 뽑기    │
│ [로봇 손/쪽지 Lottie 영역]      │ 중복 허용     │
│                               │ 순차/한 번에  │
└───────────────────────────────┴─────────────┘
```

디자인:

- 쪽지, 상자, 손, 로봇 팔 등의 밝은 교실 소품을 사용한다.
- 결과 공개 전에는 이름이 보이지 않도록 긴장감을 준다.
- 순차 공개일 때는 남은 쪽지 수를 크게 표시한다.

주요 문구:

- "쪽지를 뽑아요"
- "한 명 뽑기"
- "순서 뽑기"
- "다음 공개"

## 9.4 돌림판

목표: 학생들이 화면을 보며 기대감을 느끼는 대표 추첨 화면을 만든다.

```txt
┌───────────────────────────────┬─────────────┐
│ 큰 돌림판                       │ 설정/상태     │
│ 중앙: 시작 버튼 또는 로봇 아이콘  │ 당첨 인원     │
│ 상단: 포인터                    │ 제외 상태     │
└───────────────────────────────┴─────────────┘
```

디자인:

- 참가자 이름은 칸 안에서 잘리지 않게 줄임/회전/외곽 라벨 규칙을 둔다.
- 가중치가 있으면 칸 크기로 반영하되 설정 패널에도 설명한다.
- 스핀 중에는 라임 glow, 종료 시 로봇 축하 컷 또는 confetti를 사용한다.

## 9.5 사다리타기

목표: 역할, 청소 구역, 모둠 임무를 공정하게 배정한다.

```txt
┌───────────────────────────────┬─────────────┐
│ 사다리 보드                     │ 역할 라벨     │
│ 이름 상단                       │ 공개 방식     │
│ 결과 하단                       │ 다시 만들기   │
└───────────────────────────────┴─────────────┘
```

디자인:

- 선은 두껍고 대비 있게 표현한다.
- 현재 이동 중인 경로는 라임색으로 강조한다.
- 도착 결과는 이름과 라벨을 함께 큰 카드로 보여준다.

## 9.6 결과 화면

목표: 당첨 결과를 크고 명확하게 보여주고, 바로 수업에 활용하게 한다.

```txt
┌─────────────────────────────────────────────┐
│ 당첨!                                       │
│ [당첨자 큰 이름 카드]                         │
│ [로봇 축하 캐릭터]                            │
├─────────────────────────────────────────────┤
│ 전체 순위 / 역할 배정 / 조 배치 / 자리 배치     │
├─────────────────────────────────────────────┤
│ [당첨자 빼고 다시] [다른 게임] [복사] [저장]    │
└─────────────────────────────────────────────┘
```

디자인:

- 당첨자 이름은 화면의 첫 시선이 되게 한다.
- 결과 저장 이미지에서는 배경 장식보다 텍스트 대비를 우선한다.
- "당첨자 빼고 다시"는 주요 액션이지만 "처음으로"와 혼동되지 않게 한다.

## 9.7 저장/복원 상태

목표: 저장은 자동으로, 실패는 조용히 숨기지 않는다.

상태 문구:

| 상태 | 문구 |
|---|---|
| 저장 완료 | 자동 저장됨 |
| 방금 저장 | 자동 저장됨 · 방금 |
| 복원 완료 | 이전 작업을 불러왔어요 |
| 저장 실패 | 저장할 수 없어요 |
| 시크릿/차단 | 이 브라우저에서는 자동 저장이 제한돼요 |

## 11. 모션

### 11.1 공통 모션

| 상황 | 모션 |
|---|---|
| 버튼 hover | 살짝 떠오름 |
| 버튼 active | 눌림 |
| 카드 등장 | 120~180ms fade + y 이동 |
| 결과 등장 | pop + confetti |
| 저장 완료 | 작은 체크 표시 |

### 11.2 캐릭터 모션

- Hub: 로봇이 손을 흔들거나 체크리스트를 가리킴
- Lobby: 로봇이 게임 카드를 소개
- 제비뽑기: 로봇 팔 또는 손이 쪽지를 꺼냄
- 돌림판: 로봇이 시작 버튼을 누르는 짧은 연출
- 결과: 로봇이 전구/별/종이를 들고 축하

### 11.3 접근성

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

reduced-motion 상태에서는 뽑기 결과를 지연 없이 정적으로 보여준다.

## 13. 반응형

### Desktop / 전자칠판

- 기준: 1366 x 768 이상
- 게임 화면은 16:9 가로 구성을 우선한다.
- 학생 이름과 당첨자 이름은 가장 큰 시각 요소가 되어야 한다.

### Tablet

- Hub와 Lobby는 1~2열 혼합.
- 게임 화면은 액션 영역을 먼저, 설정은 접히는 패널로 제공한다.

### Mobile

- 명단 입력, 게임 선택, 결과 확인까지만 자연스럽게 대응한다.
- 실제 게임 진행은 가로 화면 또는 큰 화면 사용을 권장한다.

## 15. 하지 말아야 할 것

- 제품명을 `신답룰렛`으로 쓰지 않는다.
- 네온 보라 배경을 모든 화면에 과하게 깔지 않는다.
- 설정 화면을 게임 화면처럼 산만하게 만들지 않는다.
- 한글 텍스트를 이미지 안에 넣지 않는다.
- 슬롯/가챠 표현을 도박처럼 보이게 하지 않는다.
- 카드 안에 또 다른 카드들을 중첩해 복잡하게 만들지 않는다.
- 버튼마다 색을 다르게 써서 조작 우선순위를 흐리지 않는다.
